Your Android tablet, as a Mac display.

Telemachus creates a virtual desktop or streams an existing Mac display. USB uses ADB port forwarding; wireless is available for private, trusted LANs.

Project lineage

Built forward from SideScreen.

Telemachus began as a fork of SideScreen by Tran Vuong Quoc Dat and its contributors. Their MIT-licensed work established the core virtual-display, capture, encoding, ADB/TCP transport, Android decoding, wireless, and touch path.

Telemachus direction

Wired appliance behavior.

Existing-display mirroring, automatic client launch, fixed-rate pacing, bounded latency, capture recovery, clearer onboarding, and the Companion Screens identity.

A real display path, tuned for recency.

Hardware encode and decode, bounded queues, and observable performance.

Extend or mirror

Create an additional macOS desktop, or send the current main display to the tablet without stretching its aspect ratio.

Measured in the app

Live FPS, bitrate, RTT, decoder time, and dropped-frame counters make performance visible. External high-speed measurement is still required for a trustworthy glass-to-glass latency number.

Adjustable for the device

Choose resolution, refresh rate, bitrate, orientation, HiDPI, and quality. HEVC is preferred with H.264 fallback for compatible AVC-only tablets.

Touch is optional

Use taps, drag, scroll, pinch, and long press, or disable input and use the tablet strictly as an output display. Accessibility permission is needed only for input.

Know the security boundary.

USB is the recommended transport. Wireless is authenticated but currently unencrypted.

Wireless: trusted LAN only

The QR credential, video, telemetry, and touch traffic are sent without encryption. Do not expose the port publicly or use wireless mode on a shared or hostile network.

Private macOS API

Virtual-display creation uses unsupported CGVirtualDisplay APIs. A macOS update may change or break them, and App Store distribution is not expected.

Local QR scanning

Pairing codes are decoded locally with the open-source ZXing library. Telemachus does not operate an analytics, account, or cloud relay service.

Frequently asked questions

Is it plug and play?

After installing both apps and granting USB debugging and Screen Recording once, later USB reconnects can be automatic. A Mac host application is always required.

Does USB-C bandwidth guarantee 60 FPS?

No. The stream fits within USB 2.0 bandwidth at normal bitrates, but stable frame delivery also depends on capture cadence, hardware codecs, decoder scheduling, thermal limits, resolution, and backpressure.

Is wireless safe on public Wi-Fi?

No. The current bearer-token handshake is not encrypted. Use wireless only on a trusted private LAN and prefer USB.

Is every dependency documented?

The source is MIT licensed and the QR scanner uses Apache-2.0-licensed ZXing. Consult the third-party notice and review a generated dependency report before distributing a binary.
